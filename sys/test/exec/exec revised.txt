FreeBSD 5:
- imgact: creates vmspace images
- runs at_exec() in exec_new_vmspace()
- at_exec() runs execlist to insert items
- temp stores exec for vmspace in a register (replaces old with new)

NetBSD 4:
- exec_package: runs a vmcmd set to setup vmspace image (needs vmcmd)
- vmcmd: holds up to 9 vmcmd sets (i.e. the exec for vmspace) but can be extended
- exec_init() runs on startup & updates on exec_add
- link_es() prioritizes exec list
- new_vmcmd() creates memory & adds exec cmds to a list
- creates and adds to emulation (if it doesn't exist). which exec a sigcode_map (object) to map
into kernel space (for permanent reference) before copying it out to unmap it.

2.11BSDx86
- exec_linker: creates a map of vmspace image 
- exec_mmap: passes the vmspace image map through to vmspace (needs exec_linker)

- use FreeBSD vmspace_map & vmspace_unmap
- use NetBSD link_es() to prioritize
- merge exec_mmap & vmcmd: to allow the creation of a list/register of exec mmaps
